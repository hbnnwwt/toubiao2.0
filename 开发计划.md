# 开发计划

## 总体策略：Vibe Coding 优先

基于 Django + React 混合架构，最大化 AI 生成代码质量：
- Django Admin 承担内部管理 + 审批工作（AI 生成质量极高）
- React 前台只做投标方界面（客户体验）
- DRF OpenAPI 自动同步 TypeScript 类型（减少联调错误）

---

## 并行开发建议

1. **任务包 B（数据模型）** 和 **任务包 C（Admin/API）** 可流水线进行：模型完成后立即生成 Admin 和序列化器
2. **任务包 E（React 前台）** 在 C6（DRF 序列化器）完成后立即并行启动
3. **任务包 D（审批工作流）** 相对独立，可安排在 C2-C3 之后
4. **F1（JWT 认证）** 是关键路径，必须在 E1 之前完成

## 技术债务与风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| django-viewflow AI 生成质量 | 中 | 优先使用 viewflow Quick Start 模式，减少定制 |
| Ant Design 表单复杂度 | 中 | 使用 Ant Design Pro 基础模板，减少从零构建 |
| OpenAPI 类型同步失败 | 低 | 手动校验关键类型定义 |
| 多级审批逻辑变更 | 高 | 第一阶段先用两级审批（项目经理+管理员） |

## Vibe Coding 提示词模板

**后端模型（对 AI 说）：**
> "创建 BidSubmission 模型，包含：关联 Project 外键、关联提交用户、投标类型（商务标/技术标）、状态（draft/submitted/approved/rejected）、附件文件、创建时间。注册到 Django Admin。"

**前端页面（对 AI 说）：**
> "使用 Ant Design 的 ProComponents 创建投标表单页，包含：项目选择（select）、投标类型（radio）、模板选择（tree-select）、商务标富文本编辑器、技术标富文本编辑器、附件上传（Dragger）。使用 TanStack Query 提交表单。"
